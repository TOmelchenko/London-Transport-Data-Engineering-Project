# 2. Business scenario

You are working as a **junior data engineer** for a **London transport analytics team**.

The organization collects operational data from different internal systems and wants to prepare this data for reporting and analysis.

Management wants to answer questions such as:

* Which stations are the busiest?
* Which lines carry the highest passenger volume?
* Which journeys are delayed most often?
* Which transport modes show the most activity?
* Which areas of London generate the most transport traffic?

At the moment, the raw data is spread across different files and formats.
It is not ready for direct reporting.

Your job is to help organize and prepare this data in a professional way.

# 4. What makes this project realistic

In real data engineering work, data rarely arrives in one perfect clean table.

Instead, it often comes from:

* different departments
* different systems
* different file formats
* inconsistent or messy exports

That is exactly the kind of situation simulated in this project.

You will work with multiple raw files and prepare them through two different pipeline styles:

* **ETL**
* **ELT**

This is important because real data teams often need to understand both approaches.

So even though this is a guided student project, the business logic and engineering flow are realistic.
